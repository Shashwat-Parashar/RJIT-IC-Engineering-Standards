## Overview

The **RJIT Innovation Club Engineering Standards (RJIT-IC-EDS)** repository serves as the official engineering documentation repository of the RJIT Innovation Club.

It establishes a standardized framework for creating, organizing, reviewing, maintaining, and publishing engineering documentation across all club projects.

Its purpose is to promote consistency, professionalism, traceability, and long-term knowledge preservation throughout the engineering project lifecycle.

This repository provides the following resources:

- Engineering Documentation Standards
- Engineering Document Templates
- Project Proposal Templates
- Design Documentation
- Engineering Reports
- CAD Documentation Standards
- Testing Procedures
- Software Documentation
- Reference Material
- Project Examples

---

## Objectives

The objectives of this repository are to:

- Establish a unified engineering documentation standard for all Innovation Club projects.
- Standardize document creation, organization, and revision practices.
- Improve collaboration between multidisciplinary engineering teams.
- Preserve engineering knowledge across successive student batches.
- Provide reusable engineering document templates and workflows.
- Promote professional engineering documentation practices.
- Maintain a centralized technical knowledge base for the club.

## Guiding Principles

The engineering documentation framework is founded upon five core principles:

- **Clarity** – Documentation should be easy to understand and free from ambiguity.
- **Consistency** – Documents should follow standardized formats, terminology, and workflows.
- **Traceability** – Engineering decisions, revisions, and changes should be recorded and traceable throughout the project lifecycle.
- **Reproducibility** – Documentation should enable future members to understand, reproduce, and continue engineering work.
- **Professionalism** – All documentation should reflect accepted engineering practices and maintain a high standard of technical communication.
