# Security Policy

> Guidelines for responsibly reporting security concerns and protecting the integrity of the RJIT Innovation Club Engineering Standards repository.

---

## Purpose

The purpose of this Security Policy is to define how security-related issues affecting the RJIT Innovation Club Engineering Standards repository should be identified, reported, and resolved.

Maintaining the integrity, reliability, and confidentiality of repository content is a shared responsibility of all contributors.


## Scope

This policy applies to:

- Repository source files
- Engineering documentation
- Templates
- Automation scripts
- Published documents
- Repository configuration
- GitHub workflows


## Reporting a Security Issue

If you discover a security issue affecting this repository, please report it privately to the repository maintainers as soon as possible.

When reporting an issue, include:

- A clear description of the issue
- Steps to reproduce the problem, if applicable
- The affected files or components
- Any potential impact
- Suggested mitigation, if known


## Sensitive Information

The following information must never be committed to this repository:

- Passwords
- API keys
- Authentication tokens
- Private SSH keys
- Personal information without permission
- Confidential project documents
- Proprietary third-party material without authorization


## Responsible Disclosure

Contributors are encouraged to report security concerns responsibly and confidentially.

Please avoid publicly disclosing security vulnerabilities until repository maintainers have had a reasonable opportunity to investigate and address the issue.


## Repository Integrity

Contributors should help maintain repository integrity by:

- Verifying documentation before publication.
- Reviewing changes carefully.
- Maintaining accurate revision history.
- Following contribution guidelines.
- Protecting repository credentials.
- Keeping dependencies and automation scripts secure.


## Security Best Practices

Contributors are encouraged to:

- Enable Multi-Factor Authentication (MFA) on GitHub accounts.
- Use strong and unique passwords.
- Keep local development environments updated.
- Verify commits before pushing changes.
- Avoid uploading sensitive or confidential information.


## Contact

Questions or reports regarding repository security should be directed to the repository maintainers.

All reports will be reviewed responsibly and handled with appropriate confidentiality.
